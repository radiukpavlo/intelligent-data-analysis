Байєсівське середнє¶

  • Зважене середнє, яке враховує кількість наявних оцінок.
  • Доцільне, коли доступно небагато даних.
  • Широко використовується у спортивній статистиці (наприклад, для batting average).
  • Обчислюється за такою формулою:

$$r_{i} = \frac{C \times m + \Sigma{\text{ratings}}}{C+N}, $$

де:

  • $C$ – наш рівень довіри (середня кількість оцінок для заданого фільму);
  • $m$ – апріорна оцінка (глобальна середня оцінка);
  • $N$ – загальна кількість оцінок для фільму $i$.

Спочатку потрібно отримати count і mean для кожного фільму в нашому наборі даних. Це можна зробити за допомогою розширеного groupby, який одночасно застосовує mean і count через метод agg.

C = movie_stats['count'].mean()
m = movie_stats['mean'].mean()

def bayesian_avg(ratings_i):
    bayesian_avg = (C*m+ratings_i.sum())/(C+ratings_i.count())
    return bayesian_avg

  • Порожня комірка = оцінка відсутня.

from scipy.sparse import csr_matrix

def create_X(df):
    """
    Generates a sparse matrix from the ratings dataframe.
    
    Args:
        df: ratings dataframe
    
    Returns:
        X: sparse matrix
        user_mapper: dict that maps user id's to user indices
        user_inv_mapper: dict that maps user indices to user id's
        movie_mapper: dict that maps movie id's to movie indices
        movie_inv_mapper: dict that maps movie indices to movie id's
    """
    N = df['userId'].nunique()
    M = df['movieId'].nunique()

    user_mapper = dict(zip(np.unique(df["userId"]), list(range(N))))
    movie_mapper = dict(zip(np.unique(df["movieId"]), list(range(M))))
    user_inv_mapper = dict(zip(list(range(N)), np.unique(df["userId"])))
    movie_inv_mapper = dict(zip(list(range(M)), np.unique(df["movieId"])))
    
    user_index = [user_mapper[i] for i in df['userId']]
    movie_index = [movie_mapper[i] for i in df['movieId']]

    X = csr_matrix((df["rating"], (movie_index, user_index)), shape=(M, N))
    
    return X, user_mapper, movie_mapper, user_inv_mapper, movie_inv_mapper

Обчислення розрідженості матриці¶

Розгляньмо, наскільки розрідженою є наша матриця $X$. Щільність матриці $d$ можна обчислити за такою формулою:

$$ d=\frac{\text{num non-zero elements}}{\text{total elements}}. $$

In [21]:

print(f"Matrix density: {round(density*100,2)}%")

Наша матриця є досить розрідженою. Однак це типовий випадок для матриць типу «користувач-об’єкт». Загальне практичне правило полягає в тому, що щільність матриці не повинна бути нижчою за 0.5%, якщо очікується отримання прийнятних результатів.

Крок 5. Пошук подібних фільмів за допомогою k-Nearest Neighbours¶

k-Nearest Neighbours (kNN) – це алгоритм класифікації, який передбачає мітку заданого зразка на основі голосування більшості поміж його найближчих $k$ сусідів.

Відстань між двома зразками можна вимірювати за допомогою cosine similarity, Euclidean distance, Manhattan distance тощо.

Cosine Similarity та Euclidean Distance¶

  • Cosine Similarity: вимірює подібність двох точок за їхньою орієнтацією (тобто за косинусом кута між $A$ та $B$):
    • чим ближче значення cosine similarity до 1, тим подібнішими є об’єкти:

$$ \text{similairity}=\frac{A \cdot B}{\mid{A}\mid\mid{B}\mid}. $$

  • Euclidean Distance: вимірює відстань між двома об’єктами у $n$-вимірному просторі (тобто довжину прямої між $A$ та $B$):
    • на відміну від cosine similarity, Euclidean distance враховує масштаб:

$$ d(p,q) = \sqrt{(p_1-q_1)^2 + (p_2-q_2)^2 + ... + (p_n-q_n)^2}. $$

Cosine Similarity та Euclidean Distance¶

  • Пари A-B і D-E мають однакову cosine similarity.
  • Для A-B Euclidean distance менша, ніж для D-E.

Створимо функцію, яка знаходить $k$ подібних фільмів для заданого фільму.

In [24]:
from sklearn.neighbors import NearestNeighbors

def find_similar_movies(movie_id, X, k, metric='cosine', show_distance=False):
    """
    Finds k-nearest neighbours for a given movie id.
    
    Args:
        movie_id: id of the movie of interest
        X: user-item utility matrix
        k: number of similar movies to retrieve
        metric: distance metric for kNN calculations
    
    Returns:
        list of k similar movie ID's
    """
    neighbour_ids = []
    
    movie_ind = movie_mapper[movie_id]
    movie_vec = X[movie_ind]
    k+=1
    kNN = NearestNeighbors(n_neighbors=k, algorithm="brute", metric=metric)
    kNN.fit(X)
    if isinstance(movie_vec, (np.ndarray)):
        movie_vec = movie_vec.reshape(1,-1)
    neighbour = kNN.kneighbors(movie_vec, return_distance=show_distance)
    for i in range(0,k):
        n = neighbour.item(i)
        neighbour_ids.append(movie_inv_mapper[n])
    neighbour_ids.pop(0)
    return neighbour_ids

Перевіримо роботу нашої функції, передавши movieId. Створимо словник, який відображає movieId на назву фільму, щоб результати було легше інтерпретувати.

У цьому випадку movie_id = 1 відповідає Toy Story. Встановимо k = 10 і використаємо метрику за замовчуванням – cosine similarity. Це означає, що ми шукаємо 10 фільмів, найбільш подібних до Toy Story.

Факторизація матриць¶

  • Метод лінійної алгебри, який допомагає виявляти латентні ознаки між користувачами та фільмами.
  • Латентні (приховані) ознаки забезпечують компактніше подання смаків користувачів і характеристик об’єктів.
  • Може підвищити якість рекомендацій, коли матриця $X$ є дуже розрідженою.
  • Розкладає матрицю «користувач-об’єкт» на дві «факторні матриці»: – матрицю користувацьких факторів (n_users, k); – матрицю факторів об’єктів (k, n_items).

Сингулярний розклад матриці¶

  • Singular Value Decomposition (SVD) – це один із різновидів факторизації матриць, який використовують для зменшення обсягу даних і приглушення шуму
  • scikit-learn містить клас TruncatedSVD, який можна використати для зменшення розмірності нашої матриці з (n_users, n_movies) до (n_users, n_components), де n_components означає кількість латентних ознак.
In [27]:
from sklearn.decomposition import TruncatedSVD

Рекомендаційна система Top N¶

  • Факторизація матриць дає змогу передбачати відсутні оцінки у вихідній матриці $X$.
  • Відновлюємо матрицю, обчислюючи внутрішній добуток матриці користувацьких факторів і матриці факторів фільмів.

  • відсортувати передбачені оцінки за спаданням.
